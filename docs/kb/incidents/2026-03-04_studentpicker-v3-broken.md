# Инцидент: StudentPicker v3 rewrite сломал пикер групп на проде

**Дата:** 2026-03-04
**Severity:** P1 (функциональность сломана — учитель не мог назначить ДЗ группам)
**Статус:** Решён (откат к рабочей версии)

## Что произошло

2026-03-02 были сделаны 2 коммита, полностью переписывающие StudentPicker (выбор групп/учеников в конструкторе ДЗ):

- `41340dfe` — compact StudentPicker layout to prevent sidebar overflow
- `b70c59f5` — rewrite StudentPicker to modal-first design (v3)

Эти изменения (722 удалений, 524 вставок) пролились со staging на production через очередной деплой фронт-билда. В результате пикер групп перестал корректно работать.

## Затронутые файлы

- `frontend/src/modules/homework-analytics/components/homework/StudentPicker.js`
- `frontend/src/modules/homework-analytics/components/homework/StudentPicker.css`

## Причина

1. StudentPicker был полностью переписан на "modal-first design v3" — новый UI сломал функционал выбора групп
2. Изменения были сделаны на staging-ветке без тестирования
3. Попали в production-билд при очередном деплое (npm run build включает всё из рабочего дерева)

## Решение

1. Определены сломавшие коммиты: `41340dfe` и `b70c59f5` (2026-03-02)
2. Найден последний рабочий коммит: `269354b7` (2026-02-25)
3. Восстановлены оба файла: `git checkout 269354b7 -- StudentPicker.js StudentPicker.css`
4. Проверен билд: `npm run build` — успешно
5. Закоммичен откат: `aae4a998`
6. Задеплоен на прод через scp/zip
7. nginx перезагружен

## Уроки

### ПРАВИЛО: Полный rewrite компонента = отдельная ветка + тестирование

- Переписывание компонента с нуля (>500 строк изменений) НЕЛЬЗЯ делать в staging напрямую
- Нужна feature branch + manual QA + review перед merge
- StudentPicker — критический компонент (без него нельзя назначить ДЗ)

### ПРАВИЛО: Проверять функционал после деплоя

- После каждого деплоя минимальный smoke-test: "можно ли создать ДЗ и назначить группу?"
